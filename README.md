## Portfolio Visual Key

Each project card in this portfolio uses a consistent visual encoding system. Here's how to read it:

### Color — Primary Technology Stack

| Color | Level | Meaning |
|---|---|---|
| 🩵 Cyan | Profile / Master | Portfolio-level banner only — represents the synthesis of the two deepest skill areas (Python + SQL). Does not appear on individual project cards. |
| 🔵 Blue | Project | Python ecosystem (pytest, pandas) |
| 🟠 Orange | Project | JavaScript / Node ecosystem (Postman, Newman, Playwright) |
| 🟣 Purple | Project | Hybrid — Python + JavaScript together |
| 🟢 Green | Project | SQL / Data dominant |
| 🔴 Red | Project | Future expansion (Java, etc.) |

### Shading — Complexity

- **Light shade** = foundational / moderate complexity
- **Medium shade** = intermediate complexity
- **Dark / rich shade** = advanced complexity

Two projects sharing the same color means they use the same primary stack — shading distinguishes their relative depth.

### The 5 Dots — Testing Disciplines Present

Each dot represents a distinct testing discipline demonstrated in the project:

| Position | Discipline |
|---|---|
| Dot 1 | Manual / functional testing |
| Dot 2 | API testing |
| Dot 3 | Data validation |
| Dot 4 | UI automation |
| Dot 5 | CI/CD integration |

A filled dot means that discipline is present. Read the dot pattern left to right to understand what each project covers.

### Vertical Lines — System Layer Depth

- **Number of lines** = number of distinct system layers under test (API, UI, database, ETL, etc.)
- **Line thickness** = depth of testing within those layers
  - Thin = surface / smoke level coverage
  - Medium = functional coverage
  - Thick = deep validation including edge cases, business rules, and negative testing

---

*This encoding system is designed to scale as the portfolio grows. New projects will follow the same key — color, shading, dots, and lines tell the full story at a glance.*
